## 6. 결론

### 탈락: Qwen2.5-1.5B

- 0.5B와 동일 토크나이저(토큰 효율 동일)인데 크기만 2.5배
- 한국어 안정성 오히려 **더 나쁨** (95%, 중국어/영어 혼입)
- "담배 판매" 질문에 영어로 거부, "1+1 행사"에 중국어 답변
- **어떤 시나리오에서도 0.5B나 Gemma3 대비 이점이 없음**

### 최종 후보 3종

| | Qwen2.5-0.5B | Gemma3-1B | EXAONE-2.4B |
|---|:---:|:---:|:---:|
| 포지션 | 초경량 / 속도 우선 | 균형형 | 품질 우선 |
| GGUF 크기 | 379MB | 778MB | 1,569MB |
| 최소 RAM | 2GB | 2GB | 4GB |
| 응답 시간 | **2초** | 4초 | 6초 |
| 한국어 | 100% | 100% | 100% |
| 토큰 효율 | 1.45 | 1.79 | **2.20** |
| 답변 품질 | 피상적, Hallucination | 구조화, 상세 | **가장 자연스럽고 실무적** |
| 강점 | 가장 빠르고 가벼움 | 속도-품질 밸런스 | 한국어 최적화, 최고 품질 |
| 약점 | 부정확한 정보 생성 | 다소 느림 | 크기가 큼 |

### 의사결정 가이드

```
매장 POS/엣지 서버 스펙이?
  │
  ├─ RAM < 2GB ──────────── ▶ 중앙 서빙 (엣지 모델 배포 불가)
  │
  ├─ RAM 2~4GB ──────────── ▶ Qwen2.5-0.5B (379MB)
  │                            └ 2초 응답, RAG 컨텍스트 필수
  │
  ├─ RAM 4GB+ ───────────── ▶ Gemma3-1B (778MB) 또는 EXAONE-2.4B (1.5GB)
  │                            ├ 응답 시간 4초 OK → Gemma3-1B
  │                            └ 품질 우선 → EXAONE-2.4B
  │
  └─ RAM 8GB+ ───────────── ▶ EXAONE-2.4B 권장
                                └ 최고 품질 + 한국어 최적화
```

### 핵심 인사이트

1. **EXAONE-2.4B는 GGUF에서만 사용 가능** — transformers(HuggingFace)와 Ollama에서는 호환 에러 발생, llama-cpp(GGUF)로는 정상 동작
2. **Zero-shot 한계** — 모든 모델이 RAG 컨텍스트 없이는 부정확한 정보를 생성. LoRA 파인튜닝 + RAG 연동 후 재평가 필요
3. **GGUF 양자화 효과** — Q4_K_M 양자화로 transformers 대비 2~4배 속도 향상, 답변 품질은 거의 동일
